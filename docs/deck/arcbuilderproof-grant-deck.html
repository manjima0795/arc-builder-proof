Builder evidence is fragmented

Grant reviewers need proof, but builder progress lives across scattered links.

01

Offchain portfolios are easy to lose or fake

GitHub repos, demos, screenshots, docs, and social posts sit in separate systems with no shared source of truth.

02

Review cycles are slow

Teams spend time checking wallet ownership, source URLs, deployed apps, and grant evidence manually.

03

No persistent builder graph

As a builder ships more work, their Arc-native reputation should compound — not reset every application cycle.

04

MVPs lack credible provenance

Early projects need a simple way to show what is live, what contract backs it, and what evidence proves progress.

How it works under the hood

Static frontend + Arc smart contract + public proof URLs.

UI

Vite React App

Static deploy on Vercel/GitHub Pages with hash routing.

◎

User Wallet

Injected wallet signs profile and proof transactions.

SC

Arc Contract

Stores profile/proof strings, ownership, and timestamps.

↗

Reviewer Link

Public proof pages resolve by ID for grant review.

Core contract functions

upsertBuilderProfile, createProjectProof, updateProjectProof, getProjectProof, getProofIdsByBuilder.

Security posture

No private data required. Contract stores public strings only; secrets and sensitive personal data should never be published as metadata.
